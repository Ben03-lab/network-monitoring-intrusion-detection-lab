4. Viewing tcpdump Options & Analyst Use Case
Response:
You can view tcpdump options by running tcpdump --help or using the manual page with man tcpdump. Security analysts use tcpdump to capture and inspect network traffic in real time, helping them identify suspicious activity, validate IDS alerts, and troubleshoot network issues.

5. Displaying Ethernet Interfaces in Linux
Response:
A common command to list all Ethernet interfaces is:

ip link show

This is valuable to a security analyst because it helps identify active interfaces, verify configurations, detect unauthorized interfaces, and ensure the system’s network footprint matches expected security baselines.

5. Difference Between Active and Passive Scanning
Response:
Active scanning sends traffic or probes to a target to gather information, while passive scanning observes existing network traffic without interacting with the target. Active methods reveal more detail but risk detection; passive methods are stealthier but limited to naturally occurring traffic.

6. Significance of Kerberos Output
Response:
Kerberos output is significant because it shows the authentication steps taking place, including ticket requests and responses. This helps analysts verify that secure, time‑synchronized authentication is functioning correctly and can reveal issues such as failed logins, misconfigurations, or suspicious ticket activity.
